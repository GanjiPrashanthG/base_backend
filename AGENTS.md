## Learned User Preferences

- Prefer current or latest dependency versions when adding or scaffolding packages for this repo.
- Keep integration secrets (Sentry DSN, API keys, etc.) in `.env` or the deployment secret store only; never commit them or embed them in docs; rotate credentials if they are exposed.
- Mayday-api uses Express (user explicitly moved the plan off Fastify); treat Express 5 with ESM as the default HTTP stack unless direction changes.
- Avoid magic numbers and repeated literals in application code; keep shared values in a constants module (or equivalent single place).
- Use Node 26 locally via nvm (`nvm use` aligned with `.nvmrc`).
- Production deployment target is **AWS ECS Fargate** (multi-stage Docker image, ALB/`GET /health` checks).

## Learned Workspace Facts

- `mayday-api` targets Node `>=26`, native ESM (`"type": "module"`), TypeScript with NodeNext resolution, `tsx` for dev watch, and `tsc` emitting to `dist/` for production `node` runs.
- HTTP layer is Express 5 with Helmet, env-driven CORS (`CORS_ORIGIN`), compression, JSON body size limits, rate limiting, optional `trust proxy` (`TRUST_PROXY_HOPS`), Node HTTP `requestTimeout` / `headersTimeout`, readiness at `GET /health/ready` (DB ping when Mongo enabled), Prometheus **`GET /metrics`** (optional **`METRICS_REQUIRE_AUTH`** + strict admin key), **Scalar** at **`GET /api/docs`** (mounted before global Helmet), duplicate API mounts at **`/api`** and **`/api/v1`**, optional **Sentry** when **`SENTRY_DSN`** is set (`Sentry.init` in `server.ts` with optional **`SENTRY_RELEASE`**; `createApp` calls **`Sentry.setupExpressErrorHandler`** after API routes and before the JSON 404/error middleware so **`next(err)`** 5xx attach request context without duplicating `captureException` in `errorHandler`), admin-guarded **`GET …/migrations`** (Mongo migrate-mongo changelog), request IDs (`X-Request-Id` / `pino-http` `genReqId`), JSON 404 for unknown routes, centralized error handling (including 413 for oversized bodies), and structured logging via Pino (`pino-http`). Dependabot opens weekly npm update PRs (`.github/dependabot.yml`). **Husky** runs `npm run lint` on pre-commit when a `.git` repo is present; Docker and CI set **`HUSKY=0`** / **`ENV HUSKY=0`** so `npm ci` does not require hooks.
- Env loading uses `dotenv`; required settings and shapes are validated with Zod (see `src/config.ts` and `.env.example`). Production defaults to **`.env.production`** (`DOTENV_PATH` override); template **`.env.production.example`**.
- **MongoDB** uses the native **`mongodb`** driver in **`src/lib/mongo.ts`** (renamed from `prisma.ts` for **tsx**/`mongodb` module resolution; **Prisma removed**). **`MONGODB_ENABLED`** controls connect + DB routes. Schema/data migrations: **[`migrate-mongo`](https://github.com/seppevs/migrate-mongo)** (`mongo/migrations/`, `migrate-mongo-config.js`). `DATABASE_URL` can be synthesized from `MONGODB_URI` + `MONGODB_DB_NAME` + pool options via `loadConfig()`; tune **`MONGODB_MAX_POOL_SIZE`** for Fargate RAM.
- `process` handlers log `unhandledRejection` and `uncaughtException`. Graceful shutdown closes MongoDB; repeated SIGINT/SIGTERM after shutdown has started is ignored; `SHUTDOWN_GRACE_MS` caps wait then forces exit.
- Quality gates: ESLint + Prettier (flat config), `node:test` + Supertest in `test/` (`npm test` → **`node scripts/run-tests.mjs`**, Node 20+), GitHub Actions CI (SemVer check via `version:validate`, typecheck, lint, build, test, `mongo:migrate:status`, verify). A separate **`version`** workflow runs on `workflow_dispatch` and on **`v*`** tags to assert the tag matches `package.json` (npm release convention).
- Local workflow: root **`Makefile`** — **`setup`** (deps, `.env`, Mongo, migrations), **`up`** / **`down`** (compose), **`dev`** (`tsx watch`; Mongo via compose).
- `npm run verify` builds, starts the server briefly, and checks `/`, health, readiness, **metrics**, **`/api/docs`**, **`/api/v1/*`** (hello, migrations when Mongo on, db health), mirroring CI.
- Production **`Dockerfile`** targets **AWS ECS Fargate**: multi-stage build, **`npm ci --omit=dev --omit=optional`**, runner **`NODE_OPTIONS=--max-old-space-size=256`**, **`HOST=0.0.0.0`**, **`HEALTHCHECK`** on **`GET /health`**. **Task RAM (idle ~33MB** lean): Mongo driver loads only when **`MONGODB_ENABLED=1`** (dynamic import, no Prisma engine); **`API_DOCS_ENABLED`** defaults off in production; **`METRICS_ENABLED=0`** skips prom-client; **`SENTRY_DSN`** unset skips **`@sentry/node`**. **Fargate**: **512 MB** for API-only; **512–1024 MB** with Mongo under load.
